# ZiTeg / ZiChecker — фронтенд для работы с VAST

Одностраничное веб-приложение **ZiTech**: разбор рекламного **тега** (URL) и глубокая проверка **VAST XML** — без отдельного бэкенда бизнес-логики, только статика и минимальный HTTP-сервер для раздачи файлов.

## Стек

| Компонент | Назначение |
|-----------|------------|
| `index.html`, `style.css` | Разметка и стили, вкладки ZiTeg / ZiChecker / Справка |
| `app.js` | Логика ZiTeg: парсинг URL, таблица параметров, валидация, темы |
| `checker.js` | Логика ZiChecker: парсинг VAST, wrapper-цепочка, плеер, отчёты |
| `ad-infra.js` | Каталог доменов SSP/DSP и эвристики по `<AdSystem>` |
| `server.js` | Node HTTP — раздача статики (порт по умолчанию **3847**) |
| CodeMirror 5 | Редактор XML во вкладке ZiChecker (CDN) |

Проверка типов: `npm run check` (TypeScript, `jsconfig.json`).

## Запуск

```bash
npm start
```

Откройте в браузере: `http://localhost:3847/` (порт задаётся переменной окружения `PORT`).

### Docker

```bash
docker build -t ziteg .
docker run -p 3847:3847 ziteg
```

## ZiTeg (вкладка «Парсер»)

**Назначение:** разобрать **строку VAST-тега** как URL — выделить **базовый URL (BASE)** без query-строки и таблицу параметров `ключ=значение`. Запрос к рекламному серверу **не выполняется**.

**Возможности:**

- Парсинг вставленного тега; редактирование BASE и параметров в таблице (вкл/выкл строки, добавление, сортировка ключей A–Я).
- Типы значений: макросы в `{…}`, `[…]` (IAB), `%…%` (в т.ч. после URL-decode, например `%25…%25`), полные `https://…`, статика, пустые значения.
- Подсказки по распространённым макросам и именам query-параметров (GAM, OpenRTB-стиль и др.).
- Валидация структуры тега с выводом ошибок и предупреждений.
- Сборка итогового URL внизу страницы; копирование URL и варианта с кодированием.
- Блок **«Инфраструктура»:** сопоставление домена BASE и полных `https://…` в значениях параметров с внутренним каталогом (`ad-infra.js`).
- История недавних тегов в `localStorage`.
- Счётчики: параметры, макросы, отключённые строки, статус проверки.

## ZiChecker (вкладка «Валидатор»)

**Назначение:** загрузить или вставить **VAST XML**, разобрать документ, при необходимости **раскрутить цепочку wrapper’ов**, просмотреть медиа и трекеры, **проиграть** линейное видео и увидеть срабатывание событий.

**Режимы ввода:**

- **По URL** — HTTP(S)-загрузка VAST (с диагностикой).
- **По XML** — вставка XML в редактор: форматирование, проверка well-formed.

**Возможности:**

- Разбор `<VAST>`: версия, InLine / Wrapper, `AdSystem`, `AdTitle`, `Duration`, `skipoffset`, события (Impression, Error, Tracking и др.).
- Рекурсивное разрешение **wrapper-цепочки** до InLine или ошибки.
- Список **MediaFile** и нелинейных/интерактивных ресурсов; детекция VPAID/SIMID; подсказки по совместимости.
- Встроенный **видеоплеер**, выбор источника, имитация skip / click / fullscreen, лента событий по трекерам.
- Валидация по правилам IAB и распространённым требованиям SSP (обязательные элементы, длительность, длина `VASTAdTagURI`, delivery и т.д.).
- Сводка по документу, блок диагностики («почему не играет»), проверки compliance (где реализовано).
- **Инфраструктура:** те же карточки платформ по доменам из URL в ответе и по тексту `<AdSystem>`.

## Справка VAST

Вкладка **«Справка»** — встроенная документация: структура URL, макросы IAB / GAM / OpenRTB, таблицы кодов ошибок, обзор платформ РФ/СНГ и международного рынка, заметки по IMA, OMID/SIMID и др.

## Смысл названий

| Модуль | Роль |
|--------|------|
| **ZiTeg** | **Teg** — тег: разбор **URL тега** (параметры, макросы, инфраструктура по доменам). |
| **ZiChecker** | **Checker** — проверка **готового VAST XML** (цепочка, медиа, плеер, отчёты). |
